# Planning Instructions

This file intentionally mirrors the sibling instructions file in this directory. Keep both files in sync.

## Scope

Applies to `plans/`.

## Conventions

- Treat plan files as design artifacts, not implementation guarantees.
- Keep one plan per file and make the file name specific to the area being planned.
- Record decisions, assumptions, tradeoffs, open questions, and acceptance criteria rather than copying source code.
- If a plan differs from the current implementation, say so explicitly instead of implying the code already matches it.
- When code changes resolve or invalidate a plan, update the plan in the same change or mark the outdated sections clearly.
