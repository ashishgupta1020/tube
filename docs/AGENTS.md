# Documentation Instructions

This file intentionally mirrors the sibling instructions file in this directory. Keep both files in sync.

## Scope

Applies to `docs/`.

## Conventions

- Keep documentation aligned with the actual repository behavior and current product scope.
- Distinguish clearly between what is implemented now and what is planned for later.
- When describing APIs, use exact route names, field names, modes, and commands.
- If external constraints are time-sensitive, say so explicitly instead of presenting them as timeless facts.
- When code changes invalidate a doc, update the doc in the same change or remove the stale claim.
